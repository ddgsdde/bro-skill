# Shared Memory 分析器

## 任务

从原材料中提取与这个兄弟的共享记忆，构建 Shared Memory 知识库。

## 提取维度

### 1. 关系时间线
- 认识时间和方式
- 什么时候开始熟
- 关键节点（第一次一起搞事、第一次一起旅行、一起扛事、一起翻车、一起救场）
- 关系变化节点（变熟、闹翻、和好、淡了、重新联系）
- 现在的关系状态

### 2. 日常模式
- 联系频率和时间段
- 谁更主动联系
- 常见话题分布
- 线下见面频率和场景

### 3. 共同经历
- 一起去过的地方（餐厅、球场、网吧、城市、酒吧、公司、学校）
- 一起做过的事（打游戏、做项目、旅行、聚会、打球、创业、熬夜）
- 彼此印象最深的名场面
- Inside jokes / 只有你们懂的梗

### 4. 帮忙与站队
- ta帮过你什么
- 你帮过ta什么
- 真出事的时候ta通常怎么做
- 对“讲义气”“给面子”“保密”的态度

### 5. 冲突与和好
- 常见冲突原因
- ta翻脸时的典型反应
- 谁会先缓和
- 和好方式
- 冲突后关系会怎样变化

### 6. 价值观线索
- ta对钱、面子、承诺、规则、人情的态度
- ta最看不上的行为
- ta最欣赏别人的点

## 输出格式

```markdown
## 关系时间线
- 认识：{时间} {方式}
- 熟起来：{时间} {契机}
- 当前状态：{状态}

## 日常模式
{描述}

## 共同经历
{按时间线列出关键经历}

## Inside Jokes
{只有你们懂的梗和暗号}

## 互相帮忙
{关键帮助与站队记录}

## 冲突与和好
{典型冲突剧本与修复模式}

## 价值观线索
{钱、面子、承诺、边界等线索}
```

## 注意事项

- 聊天记录中的事实优先于用户口述
- 既保留高光，也保留摩擦和不舒服的部分
- 注意提取“反复出现”的模式，而不是一次性事件
- 不要使用恋爱关系术语
