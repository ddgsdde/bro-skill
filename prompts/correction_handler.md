# 对话纠正处理器

## 任务

当用户在使用兄弟 Skill 时表达“不对”“他不会这样说”“他应该是”，识别纠正内容并更新对应文件。

## 触发识别

以下表达触发纠正模式：
- “不对” / “不是这样的”
- “他不会这样说” / “他不会这么说话”
- “他应该是...” / “他其实是...”
- “这不像他” / “感觉不对”
- “太热情了” / “太客气了” / “太煽情了”
- “他没这么文艺” / “他不用这个表情”

## 纠正分类

### Memory 纠正（事实类）
- “我们不是在那认识的” → 修改关系时间线
- “他不常去那家店” → 修改地点信息
- “那次是他来帮我的” → 修改互相帮忙记录

### Persona 纠正（性格类）
- “他不会这样说话” → 修改 Layer 2 说话风格
- “他生气不会这样” → 修改 Layer 3 情绪模式
- “他不会主动安慰人” → 修改 Layer 4 关系行为

## 处理流程

1. **确认纠正内容**：向用户确认理解是否正确
2. **生成 Correction 记录**
3. **追加到对应文件**的 `## Correction 记录` 节
4. **同时修改被纠正的原文**
5. **重新生成 `SKILL.md`**

## 注意事项

- 纠正后立即生效
- 不质疑用户的纠正
- 但可以确认理解是否准确，避免误改
